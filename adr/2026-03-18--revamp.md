# ADR: Minimalist CLI Secret Manager (`pw`)

## 1. Status
**Proposed** (Refining from `denv` Go implementation)

## 2. Context
We need a local-first, cross-platform, and Git-friendly secret manager. 
* **Issues with existing tools**: `gopass` leaks metadata (filenames); `KeePass` is a binary blob that hinders Git diffs and has slow CLI cold-starts.
* **Goals**: Metadata privacy (obfuscated filenames), per-entry Git granularity, high-performance environment injection, and zero-leakage (in-memory processing).

## 3. Decision

### A. Storage Architecture
* **Encryption Engine**: `age` (v1+).
* **Physical Layer**: Each secret is a standalone file with a **randomized filename** (UUID/Hash) to prevent metadata leakage in cloud backups.
* **Indexing**: A local encrypted cache `index.dat.age` maps `__id` to physical filenames.
    * **Self-Healing**: The index is non-authoritative. It can be rebuilt by decrypting the header of all files in the vault.

### B. Data Schema (YAML + Payload)
Each secret is a YAML file, optionally followed by a raw payload separated by `---`.
```yaml
__id: github-enterprise
__tags: [work, dev]
_base_url: "https://github.company.com"
GITHUB_TOKEN: "ghp_internal_key"
API_ENDPOINT: "$_base_url/api/v3"
---
<Optional Raw Payload: SSH Keys, Certificates, etc.>
```

### C. Naming Conventions (The Prefix Rule)
* **`__xxx` (Internal)**: Reserved for tool metadata (e.g., `__id`). Not injected into shell environments.
* **`_xxx` (Local)**: Private variables for the current file. Used for `$expansion` but **not** injected into the shell.
* **`xxx` (Export)**: Standard environment variables. Automatically injected during `run`.

### D. Features
* **Variable Expansion**: Supports single-pass `$xxx` substitution within the same file or merged context.
* **Multi-ID Injection**: `pw run` allows multiple IDs. Keys from later IDs override earlier ones.
* **Conflict Resolution**: If multiple files share the same `__id`, the indexer appends a suffix (e.g., `name (1)`) based on file creation/Git commit timestamps.

## 4. CLI Specification

| Command | Usage | Description |
| :--- | :--- | :--- |
| **`ls`** | `pw ls` | List all indexed `__id`s. |
| **`show`** | `pw show <id>` | Decrypt and print the full content to stdout. |
| **`edit`** | `pw edit <id>` | Decrypt to memory, edit via `$EDITOR`, and re-encrypt. |
| **`run`** | `pw run id1 id2 -- cmd` | Inject merged envs and execute command. |
| **`mv`** | `pw mv <id> <new_id>` | Update the `__id` field and refresh index. |
| **`rm`** | `pw rm <id>` | Delete the physical file and update index. |
| **`reindex`**| `pw reindex` | Force rebuild `index.dat.age` from scratch. |
| **`rcp`** | `pw rcp <add/rm/ls>` | Manage `age` recipients (public keys). |
| **`import`** | `pw import <dir>` | Bulk import secrets from a directory. |
| **`export`** | `pw export <dir>` | Decrypt and export all secrets to a directory. |

## 5. Consequences
* **Security**: Filenames are obfuscated. Plaintext never hits the disk (piped through memory).
* **Git Efficiency**: Each change creates a distinct, small encrypted diff.
* **Simplicity**: No complex `extends` logic; reuse is handled via multi-ID injection at runtime.
